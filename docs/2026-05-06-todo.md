## 절대 하지 말 것

- **`verdict_rules.py` 에 인기도 / 다운로드 수 / 패키지 나이 다시 끌어들이기.**
  `src/pkgsentinel/verdict_rules.py:1-7` 의 docstring 에 명시된 설계 원칙
  ("패키지 나이 / 인기도 / 다운로드 수 일절 참조하지 않음"). 이번 세션에서
  popular×benign 룰을 코어로 이식했다가 이 원칙 위반으로 revert. 사용자가
  명확히 "유명 패키지가 공격당했을 때도 우리 도구가 탐지해야 한다" 는
  입장 표명. eval 스크립트 (`scripts/eval_real.py`) 안의 popular 룰은
  보존하지만, 코어 verdict 로직에 들이지 말 것.

- **`stub` 모드 단독을 production 권장으로 표시.** stub 단독은 인기
  legit 패키지 8/9 를 SUSPICIOUS+ 로 오분류 (직전 측정). LLM 모드와
  직렬로 결합되어야 의미 있음. CLI 기본값은 `claude`. stub 은
  에어갭/오프라인/회귀 테스트 한정 명시 옵트인.
